# P0-1: 自动调度闭环 - 实现总结

实施日期：2026-06-08

## 已完成内容

### 1. 数据库表设计

创建了 `strategy_cycle_runs` 表用于记录策略闭环执行状态：

```sql
CREATE TABLE IF NOT EXISTS strategy_cycle_runs (
    run_id TEXT PRIMARY KEY,
    trigger_type TEXT NOT NULL,           -- manual/scheduled/auto
    status TEXT NOT NULL,                  -- running/completed/failed
    started_at TEXT NOT NULL,
    finished_at TEXT,
    total_strategies INTEGER DEFAULT 0,
    signal_runs INTEGER DEFAULT 0,
    health_checks INTEGER DEFAULT 0,
    optimization_jobs INTEGER DEFAULT 0,
    error TEXT,
    report_json TEXT
)
```

### 2. 策略闭环调度服务

创建了 `StrategyCycleService` (`backend/app/services/strategy_cycle.py`)，实现：

- **闭环执行流程**：
  1. 回填所有活跃策略的信号前向收益
  2. 对每个策略运行选股生成当日信号
  3. 计算策略健康度
  4. 对衰减策略（health_score < 70 或 status 为 degraded/failing）自动触发优化

- **执行状态管理**：
  - `run_cycle(trigger_type)`: 执行完整闭环
  - `get_latest_run()`: 获取最近一次执行记录
  - `get_run_history(limit)`: 获取历史记录
  - `is_running()`: 检查是否正在执行
  - `get_progress()`: 获取实时进度

- **容错机制**：
  - 单个策略失败不影响其他策略执行
  - 详细记录每个策略的执行结果和错误信息
  - 闭环失败不影响下一次运行

### 3. 自动触发集成

修改了 `SyncSchedulerService` (`backend/app/services/sync_scheduler.py`)：

- 在数据同步成功完成后，自动在后台线程触发策略闭环
- 使用独立线程避免阻塞主调度循环
- 检查是否已有闭环在执行，避免重复触发

### 4. API 端点

创建了策略闭环 API (`backend/app/api/strategy_cycle.py`)：

- `POST /api/v1/strategy-cycle/trigger` - 手动触发闭环
- `GET /api/v1/strategy-cycle/status` - 获取当前状态和进度
- `GET /api/v1/strategy-cycle/history` - 获取执行历史
- `GET /api/v1/strategy-cycle/runs/{run_id}` - 获取指定执行详情

### 5. 策略监控增强

在 `strategy_monitor.py` 中添加：

- `run_strategy_signals(strategy_id)`: 运行策略选股的便捷函数
- `should_trigger_optimization(health)`: 判断是否应触发优化的决策函数

## 验收标准达成情况

✅ **用户可以在页面点击"执行完整闭环"**
   - API 端点 `/api/v1/strategy-cycle/trigger` 已就绪

✅ **系统可以在每日同步后自动运行完整闭环**
   - 已集成到 `sync_scheduler` 的数据同步成功回调中

✅ **每次闭环执行都有运行状态、耗时、错误、策略数量、生成信号数、优化任务数**
   - 所有信息都记录在 `strategy_cycle_runs` 表中
   - 提供详细的 JSON 报告包含每个策略的执行细节

✅ **闭环失败不会影响下一次运行**
   - 使用独立的 run_id 标识每次执行
   - 异常被捕获并记录，不会传播到调度器

## 使用方式

### 手动触发闭环

```bash
curl -X POST http://localhost:8000/api/v1/strategy-cycle/trigger \
  -H "Content-Type: application/json" \
  -d '{"trigger_type": "manual"}'
```

### 查询闭环状态

```bash
curl http://localhost:8000/api/v1/strategy-cycle/status
```

### 查询执行历史

```bash
curl http://localhost:8000/api/v1/strategy-cycle/history?limit=10
```

## 技术亮点

1. **并发安全**：使用 `_current_run_id` 标记避免并发执行
2. **实时进度**：通过 `_progress` 字典提供细粒度进度信息
3. **详细日志**：每个策略的执行结果都被记录到 `report_json`
4. **异步集成**：使用后台线程避免阻塞数据同步流程
5. **幂等性**：可重复执行，不会产生副作用
