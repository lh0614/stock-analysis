## 已完成内容

### 1. 多维度健康度评分模型

创建了新的健康度评分模块 `health_scoring.py`，实现结构化的多维度评分体系：

#### 子分项定义

| 维度 | 权重 | 评分范围 | 说明 |
|------|------|----------|------|
| **信号活跃度** | 10% | 0-100 | 基于近期信号数量，期望30个/月 |
| **信号成熟度** | 10% | 0-100 | 已有收益数据的信号占比 |
| **胜率** | 25% | 0-100 | 相对基准胜率的表现 |
| **收益率** | 25% | 0-100 | 相对基准收益率的表现 |
| **回撤控制** | 15% | 0-100 | 最大回撤相对基准的表现 |
| **数据质量** | 10% | 0-100 | A/B/C/D等级映射为100/80/50/20分 |
| **市场适配度** | 5% | 0-100 | 基于连续亏损次数判断 |

#### 评分逻辑

**信号活跃度（signal_activity）**
- 30个信号 → 100分（满分）
- 15个信号 → 50分
- 0个信号 → 0分
- 线性比例计算，达到期望值即满分

**信号成熟度（signal_maturity）**
- 已成熟信号数 / 总信号数 × 100
- 用于判断评估可信度
- < 50分时降低整体置信度

**胜率（win_rate）**
- ≥ 基准胜率 → 70-100分（超过越多分数越高）
- < 基准胜率 → 0-70分（按比例递减）
- 默认基准：50%

**收益率（return_score）**
- 正收益且 ≥ 基准 → 70-100分
- 正收益但 < 基准 → 50-70分
- 负收益 → 0-50分（-10%为最差）
- 默认基准：5%

**回撤控制（drawdown）**
- ≤ 基准回撤 → 70-100分
- > 基准回撤 → 0-70分（超过越多扣分越多）
- 默认基准：15%

**数据质量（data_quality）**
- A级 → 100分
- B级 → 80分
- C级 → 50分
- D级 → 20分

**市场适配度（market_fit）**
- 0次连续亏损 → 100分
- 1-2次 → 80分
- 3-4次 → 60分
- 5-6次 → 40分
- ≥7次 → 20分

### 2. 结构化健康度模型

定义了新的 Pydantic 模型：

```python
class HealthSubScores(BaseModel):
    """各子分项"""
    signal_activity: float
    signal_maturity: float
    win_rate: float
    return_score: float
    drawdown: float
    data_quality: float
    market_fit: float

class HealthScoreDetail(BaseModel):
    """详细健康度"""
    health_score: float              # 总分 0-100
    status: str                      # healthy/degraded/failing
    sub_scores: HealthSubScores      # 子分项
    degradation_reasons: List[str]   # 衰减原因列表
    recommended_actions: List[str]   # 推荐操作列表
    confidence_level: str            # high/medium/low
    last_updated: str                # 更新时间
```

### 3. 智能诊断与建议

**衰减原因自动识别：**
- 信号活跃度 < 50 → "信号活跃度较低（X分）"
- 信号成熟度 < 50 → "信号成熟度不足（X分），需等待更多数据"
- 胜率 < 50 → "胜率偏低（X分）"
- 收益率 < 50 → "收益率不佳（X分）"
- 回撤 < 50 → "回撤控制较差（X分）"
- 数据质量 < 70 → "数据质量X级，影响评估可信度"
- 市场适配 < 50 → "市场环境不适配（X分）"

**推荐操作自动生成：**
- 信号活跃度低 → "放宽筛选条件以增加信号数量"
- 胜率低 → "优化入场条件以提高信号质量"
- 收益率低 → "调整仓位管理和退出规则"
- 回撤大 → "收紧止损以控制回撤"
- 数据质量差 → "刷新数据源以提高数据质量"
- 市场不适配 → "评估市场环境变化，考虑暂停策略"
- 健康 → "策略表现健康，继续观察"

### 4. 置信度评估

**置信度等级：**
- **High**：数据质量 ≥ 80 且 信号成熟度 ≥ 70
- **Medium**：数据质量 ≥ 60 且 信号成熟度 ≥ 50
- **Low**：其他情况

置信度用于提示用户评估结果的可靠程度。
