## 📝 需求说明

在策略闭环的优化触发环节，需要将 `optimize_strategy` 的结果保存到数据库，以便：
1. 记录优化历史
2. 供用户审核优化候选版本
3. 追溯优化决策过程

---

## ✅ 实现内容

### 修改文件
`backend/app/services/strategy_cycle.py`（约 286-340 行）

### 核心改动

```python
# 1. 导入 save_optimization_result 函数
from app.services.strategy_library import save_optimization_result

# 2. 执行优化
optimization_result = loop.run_until_complete(
    optimize_strategy(spec, config)
)

# 3. 处理 candidate_spec
# - rejected 时为空字典 {}，转为 None
# - promoted 时保持原有的 dict 数据
candidate_spec = optimization_result.candidate_spec if optimization_result.candidate_spec else None

# 4. 保存到数据库
saved_optimization_id = save_optimization_result(
    result=optimization_result,
    candidate_spec=candidate_spec
)

# 5. 记录保存结果
optimization_jobs += 1
strategy_detail["optimization"] = {
    "job_id": saved_optimization_id,
    "optimization_id": optimization_result.optimization_id,
    "status": optimization_result.decision,
    "decision": optimization_result.decision,
    "reason": optimization_result.decision_reason
}
```

## 🎯 关键逻辑

### candidate_spec 处理

**问题背景**：
- `OptimizationResult.candidate_spec` 的类型是 `Dict[str, Any]`
- rejected 时为空字典 `{}`（不是 None）
- `save_optimization_result` 接受 `dict | None`

**解决方案**：
```python
# 使用 truthiness 判断
candidate_spec = optimization_result.candidate_spec if optimization_result.candidate_spec else None

# 结果：
# - {} (空字典) -> None
# - {'name': 'test'} (非空字典) -> {'name': 'test'}
```

### 两种场景处理

| 场景 | decision | candidate_spec | 转换后 | 数据库存储 |
|------|----------|----------------|--------|-----------|
| **rejected** | 'rejected' | `{}` | `None` | NULL 或 {} |
| **promoted** | 'promoted' | `{...}` | `{...}` | JSON 数据 |

## 🔄 完整流程

### 策略闭环 - 优化环节

```
1. 检测策略衰减
   ↓
2. should_trigger_optimization(health) = True
   ↓
3. 获取策略规格 (get_strategy)
   ↓
4. 构造优化配置 (OptimizationConfig)
   ↓
5. 执行优化 (optimize_strategy)
   ↓
6. 处理 candidate_spec (空字典 -> None)
   ↓
7. 保存优化结果 (save_optimization_result)
   ↓
8. 记录执行详情
   ↓
9. 继续下一个策略
```

## 🎯 使用场景

### 场景 1: 优化成功（promoted）

```python
OptimizationResult(
    decision='promoted',
    candidate_spec={
        'name': '优化后的策略',
        'version': '1.0.1',
        'entry_conditions': [...]
    }
)
↓
保存到数据库，status='pending'
↓
用户审核，可以 promote 或 reject
```

### 场景 2: 优化失败（rejected）

```python
OptimizationResult(
    decision='rejected',
    decision_reason='未找到更优参数',
    candidate_spec={}  # 空字典
)
↓
转换为 None
↓
保存到数据库，status='rejected'
↓
用户可查看失败原因，无需审核
```
