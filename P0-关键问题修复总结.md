### 问题 2: 异步函数在同步代码中调用

**严重程度**：🔴 高危（运行时错误）

**问题表现**：
- 即使修正参数，调用仍会失败
- 返回协程对象而非结果

**根本原因**：
- `optimize_strategy` 是 `async def` 定义的异步函数
- 在 `_execute_cycle()` 同步方法中直接调用
- 缺少 `await` 或事件循环处理

**影响范围**：
- `backend/app/services/strategy_cycle.py`
- 异步/同步混用导致运行时错误

## ✅ 修复方案

### 修复代码

**文件**：`backend/app/services/strategy_cycle.py`  
**位置**：约 286-303 行

```python
# 修复后的代码
if should_trigger_optimization(health):
    self._progress["sub_phase"] = "生成优化候选"
    try:
        # 1. 获取策略规格
        strategy_data = get_strategy(strategy_id)
        if strategy_data:
            from app.services.strategy_optimizer import optimize_strategy, OptimizationConfig
            from app.models.strategy_spec import StrategySpec
            import asyncio

            # 2. 构造策略规格对象
            spec = StrategySpec(**strategy_data['spec'])
            
            # 3. 构造优化配置对象
            config = OptimizationConfig(
                objective="sharpe",
                search_method="grid",
                max_trials=30
            )

            # 4. 获取或创建事件循环
            try:
                loop = asyncio.get_event_loop()
            except RuntimeError:
                loop = asyncio.new_event_loop()
                asyncio.set_event_loop(loop)

            # 5. 在同步代码中执行异步函数
            optimization_result = loop.run_until_complete(
                optimize_strategy(spec, config)
            )

            optimization_jobs += 1
            strategy_detail["optimization"] = {
                "job_id": optimization_result.optimization_id,
                "status": optimization_result.decision
            }
    except Exception as opt_err:
        print(f"优化策略 {strategy_id} 失败: {opt_err}")
        strategy_detail["optimization"] = {
            "error": str(opt_err)
        }
```
