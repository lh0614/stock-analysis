## 自动化流程验证

### 数据同步后自动触发策略闭环

1. 启用定时同步：
```bash
curl -X POST http://localhost:8000/api/v1/sync-scheduler/config \
  -H "Content-Type: application/json" \
```

4. 查看闭环执行结果：
```bash
curl http://localhost:8000/api/v1/strategy-cycle/status
```

## 常见问题

### Q1: 闭环执行失败怎么办？

查看错误详情：
```bash
curl http://localhost:8000/api/v1/strategy-cycle/runs/{run_id}
```

检查 `error` 字段和 `report_json` 中每个策略的错误信息。

### Q2: 为什么健康度置信度是 low？

低置信度通常由以下原因导致：
- 数据质量 < 60 分
- 信号成熟度 < 50 分（信号数量不足或尚未有收益数据）

解决方法：
- 刷新数据源提高数据质量
- 等待更多信号成熟（需要时间累积）

### Q3: 数据质量等级是 D，怎么改善？

D 级表示数据缺失严重：
1. 检查股票代码是否正确
2. 手动触发数据同步
3. 查看数据同步日志排查问题

### Q4: 如何查看策略优化触发的候选版本？

```bash
curl http://localhost:8000/api/v1/strategy-optimizer/jobs?strategy_id={strategy_id}
```

查看 `status=pending` 的优化记录。
