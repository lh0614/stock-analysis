## 已知限制

### 1. 数据相关
- **数据陈旧**: 当前数据更新到 2026-05-21，距今16天
- **数据源**: 仅使用baostock单一数据源
- **影响**: 单股分析和策略回测的时效性受影响

### 2. 功能限制
- **没有实时数据**: 不支持盘中实时分析
- **历史数据要求**: 需要至少60个交易日的数据
- **策略优化时间**: 参数搜索可能需要较长时间
- **健康度历史**: 未持久化，重启后丢失

### 3. 性能限制
- **批量分析**: 同时分析多只股票可能较慢
- **优化任务**: 不支持后台异步任务队列
- **数据缓存**: 未实现因子计算结果缓存

## 使用建议

### 首次使用流程

1. **更新数据** (可选但推荐)
   ```bash
   cd backend
   python3 scripts/sync_data.py  # 如果有此脚本
   ```

2. **创建测试策略**
   - 访问"智能选股"页面
   - 设置简单条件（如：RSI6 < 30, 放量突破）
   - 运行选股和回测
   - 保存评级为B或以上的策略

3. **测试单股分析**
   - 访问"单股深度分析"页面
   - 输入常见股票代码（600519, 000001, 600036）
   - 查看分析结果和证据链

4. **测试策略优化**
   - 访问"策略库"页面
   - 对已保存的策略点击"优化"
   - 等待优化完成（可能需要几分钟）
   - 查看参数变更建议

5. **测试策略监控**
   - 对活跃策略点击"健康度"
   - 查看评分和建议
   - 注意：首次使用可能缺少历史基准

### 生产使用建议

1. **数据准备**
   - 定期更新历史数据（建议每日）
   - 确保至少有60个交易日的数据
   - 监控数据质量，及时处理异常

2. **策略管理**
   - 先在测试环境验证策略
   - 逐步增加策略复杂度
   - 定期回测和优化
   - 关注健康度变化

3. **性能优化**
   - 避免同时优化多个策略
   - 限制参数搜索空间
   - 使用随机搜索而非网格搜索
   - 考虑增加因子计算缓存

## 故障排除

### 问题：单股分析返回"数据不足"
**可能原因**:
- 股票代码不存在
- 历史数据不足60天
- 数据质量等级为D

**解决方法**:
- 检查股票代码是否正确
- 更新历史数据
- 选择其他股票测试

### 问题：策略优化时间过长
**可能原因**:
- 参数组合过多
- 使用了网格搜索
- 回测数据量大

**解决方法**:
- 减少 max_trials（如改为20-30）
- 使用 random 搜索方法
- 缩小参数范围

### 问题：健康度评分不准确
**可能原因**:
- 策略刚创建，没有历史数据
- 缺少历史基准指标
- 最近没有产生信号

**解决方法**:
- 等待策略运行一段时间
- 手动运行几次选股积累数据
- 查看具体的衰减信号判断
