# 本地股票分析平台使用文档

更新日期：2026-06-06

## 1. 产品说明

Local Stock Analysis（LSA）是一个本地优先的股票分析工作台，主要用于 A 股标的的行情查看、分析流水线、选股、回测、交易计划、模拟组合、复盘和预警。

本项目不接入券商柜台，不提供实盘下单、委托、撤单、资金划转等能力。页面中的分析方向、预测区间、回测、模拟组合、计划和复盘均为研究辅助，不构成投资建议。

## 2. 当前具备的功能

### 2.1 分析与行情

- 股票代码查询。
- 自选股管理。
- 日 K 线图、均线、成交量。
- 股票摘要、历史数据、技术指标。
- 分析流水线：采集、质检、特征、策略、预测、方向、呈现。
- 短线、中线、长线方向卡：偏多、偏空、震荡、置信度、依据链、反证。
- 支撑压力与价格结构。
- 短/中/长预测区间。
- 资讯时间线。
- 分析运行历史和断点续跑。

### 2.2 数据与质量

- 股票列表同步。
- 日 K 全量/增量同步。
- 定时同步配置。
- 本地数据状态与覆盖率。
- 数据质量等级。
- 跨源冲突和缺失 K 线查看。
- 本地数据仓相关能力：DuckDB、Parquet、SQLite。

### 2.3 策略与选股

- 内置选股预设。
- 多预设交集扫描。
- 自定义股票池扫描。
- 仅本地 K 线扫描。
- 扫描过程流式进度。
- 命中结果可跳转分析、加入自选、发起回测。
- 工作流模板管理：短线、中线、长线、自定义。
- 策略工坊：策略列表、上传、版本修订、参数保存。
- 智能选股：自然语言要求解析为标准策略规格，运行选股并保存为策略。
- 策略库：查看策略健康度、真实信号、健康趋势、自进化优化记录和版本快照。
- 策略自进化：健康衰减时生成优化候选，人工晋级或拒绝。
- 因子目录和因子重算 API。

### 2.4 回测与复盘闭环

- 回测中心：股票组合、策略选择、调仓周期、市场过滤。
- 回测运行记录。
- 回测指标：总收益、最大回撤、夏普等。
- 权益曲线简表。
- 交易明细。
- 交易计划：新建计划、保存触发价和失效价、绑定预警。
- 模拟组合：手工记录模拟买入/卖出，查看持仓和成交记录。
- 复盘中心：按计划记录盈亏、标签、总结，查看累计统计。

### 2.5 预警与系统

- 价格、涨跌幅、RSI 等预警规则。
- 预警触发事件列表。
- 后台每 300 秒轮询一次预警。
- 数据源顺序配置。
- AI 解读开关。
- 缓存统计和清理。
- Swagger API 文档。

## 3. 运行方式

### 3.1 环境要求

后端：

- Python 3.11+
- 可访问公网，用于拉取行情、股票列表和资讯

前端：

- Node.js 18+
- pnpm 或 npm，推荐 pnpm

### 3.2 本地开发启动

先启动后端：

```bash
cd backend
python -m venv venv
source venv/bin/activate
pip install -r requirements.txt
python start_server.py
```

后端默认地址：

```text
http://localhost:8000
```

再启动前端：

```bash
cd frontend
pnpm install
pnpm dev
```

前端默认地址：

```text
http://localhost:5173
```

开发模式下，前端 `/api` 会代理到 `http://localhost:8000`。

### 3.3 Docker 启动

在项目根目录执行：

```bash
docker compose up --build
```

启动后访问：

```text
前端：http://localhost:5173
后端：http://localhost:8000
```

Docker 模式下后端数据目录挂载到 Docker volume `lsa-data`。

### 3.4 API 文档

后端启动后可访问：

```text
Swagger：http://localhost:8000/docs
ReDoc：http://localhost:8000/redoc
健康检查：http://localhost:8000/health
业务健康检查：http://localhost:8000/api/v1/health
```

## 4. 首次使用流程

建议按以下顺序初始化：

1. 启动后端和前端。
2. 打开 `http://localhost:5173`。
3. 进入“数据同步”页面。
4. 先同步股票列表。
5. 首次使用建议执行一次全量 K 线同步；如果只想快速体验，可先在驾驶舱查询单个股票。
6. 进入“数据质量”页面，查看质量等级、缺失和冲突。
7. 回到“分析驾驶舱”，输入股票代码开始分析。
8. 如果需要批量选股，进入“选股器”运行扫描。
9. 如果需要验证策略，进入“回测中心”运行回测。
10. 如果要做复盘闭环，创建交易计划、绑定预警、记录模拟成交并填写复盘。

首次全量同步全市场 K 线耗时较长，取决于网络、数据源稳定性和机器性能。中途断开后可继续同步。

## 5. 页面使用说明

### 5.1 分析驾驶舱

路径：

```text
/
```

用途：

- 查询单只股票。
- 查看自选股。
- 运行分析流水线。
- 查看方向、支撑压力、预测、资讯、K 线、指标和运行历史。
- 保存计划草稿。

推荐操作：

1. 在“股票查询”输入 6 位股票代码，例如 `600519`。
2. 点击“查询”。
3. 在顶部工具栏选择工作流和策略。
4. 点击“运行分析”。
5. 优先查看数据质量、市场环境、分析方向、计划草稿和支撑压力。
6. 再查看 K 线、指标、资讯、流水线详情和运行历史。

注意事项：

- 如果数据质量为 D 级，系统会阻断方向结论。
- 分析方向只是研究提示，不代表买卖建议。
- 后端未启动时，页面会显示后端离线提示。

### 5.2 选股器

路径：

```text
/screener
```

用途：

- 按预设条件批量扫描股票池。
- 支持板块过滤、排除 ST、自定义股票池。
- 支持仅使用本地 K 线，避免扫描时频繁访问外网。
- 命中结果可跳转分析、加入自选、运行回测。

推荐操作：

1. 先确认本地股票库和 K 线同步状态。
2. 勾选一个或多个选股预设。
3. 选择扫描范围。
4. 开启“仅本地 K 线”以提高稳定性。
5. 点击“开始扫描”。
6. 在命中结果中选择“分析”“回测”或“自选”。

说明：

- 多选预设当前为交集逻辑，即必须同时满足所有勾选条件。
- 如果开启仅本地 K 线，未缓存的股票会被跳过。

### 5.3 市场环境

路径：

```text
/market
```

用途：

- 查看整体市场状态。
- 查看指数趋势。
- 查看市场宽度，如上涨家数、下跌家数、下跌占比。

建议用法：

- 在单票分析前先查看市场状态。
- 当市场状态偏弱或风险较高时，降低对个股短线信号的信任度。

### 5.4 数据质量

路径：

```text
/data-quality
```

用途：

- 查看数据质量等级分布。
- 查询单只股票的数据质量。
- 查看跨源冲突、异常和缺失 K 线。

质量等级含义：

| 等级 | 含义 | 建议 |
|---|---|---|
| A | 数据较完整，多源一致 | 可正常分析 |
| B | 有轻微问题 | 可分析，注意提示 |
| C | 单源或轻微异常 | 仅作参考，降低置信度 |
| D | 缺失或冲突严重 | 不建议生成方向结论 |

### 5.5 数据同步

路径：

```text
/sync-scheduler
```

用途：

- 配置每日定时同步。
- 选择增量或全量同步模式。
- 手动立即同步。
- 查看同步状态、进度、最近结果和错误。

同步模式：

| 模式 | 说明 | 适用场景 |
|---|---|---|
| 增量 | 只补已有全量文件的落后日 K | 日常更新 |
| 全量 | 包含列表和全量 K 线断点续传 | 首次初始化或修复数据 |

建议：

- 日常盘后使用增量同步。
- 首次使用或数据大量缺失时使用全量同步。

### 5.6 回测中心

路径：

```text
/backtests
```

用途：

- 对股票组合或策略运行回测。
- 查看回测指标、权益曲线、交易明细和历史运行记录。

当前可配置项：

- 股票代码列表。
- 策略：组合等权、MA 交叉。
- 调仓周期：日、周、月。
- 市场过滤开关。

推荐操作：

1. 输入多个股票代码，例如 `600519,000001,300750`。
2. 选择策略和调仓周期。
3. 根据需要开启市场过滤。
4. 点击“运行回测”。
5. 查看总收益、最大回撤、夏普、权益曲线和交易明细。

注意：

- 回测不代表未来收益。
- 回测结果依赖本地历史数据质量。
- 若数据缺失较多，应先同步数据并检查数据质量。

### 5.7 工作流记忆

路径：

```text
/workflows
```

用途：

- 管理短线、中线、长线工作流模板。
- 设置默认工作流。
- 配置指标和图表周期。
- 导入/导出工作流。

内置工作流：

- 短线攻坚。
- 中线波段。
- 长线价值。

### 5.8 策略工坊

路径：

```text
/strategies
```

用途：

- 查看内置和自定义策略。
- 上传 `.py` 或 `.zip` 策略包。
- 查看策略版本和修订历史。
- 调整策略参数。

策略包约定：

- Python 策略需提供 `run(ctx)` 入口。
- 策略执行有超时和沙箱限制。
- 上传策略前应确认文件来源可信。

### 5.9 智能选股与策略库

路径：

```text
/intelligent-screener
/strategy-library
```

用途：

- 将中文选股要求解析为 `StrategySpec`。
- 按策略规格运行智能选股。
- 对选股结果做样本内/样本外回测。
- 将有效选股结果保存为策略。
- 查看策略健康度、近期信号、5日/20日前向收益。
- 查看健康度历史趋势。
- 查看自进化优化记录、参数变化和候选版本状态。
- 晋级或拒绝优化候选版本。
- 查看策略版本快照和关键差异。

推荐流程：

1. 进入“智能选股”，输入选股要求，例如“找回踩 MA20 不破、量能放大、RSI 不超买的股票”。
2. 点击“解析要求”，检查系统生成的股票池、入场条件、排序规则、仓位规则。
3. 点击“运行选股”，查看候选股票、命中条件和数据质量。
4. 对选股运行做回测，评级达到要求后保存为策略。
5. 进入“策略库”，将策略启用为 `active`。
6. 点击“健康度”，查看健康评分、信号、健康趋势和优化记录。
7. 如果策略衰减，点击“一键生成优化候选”。
8. 对 pending 优化任务选择“晋级”或“拒绝”。
9. 使用“执行完整监控闭环”手动运行：活跃策略信号、前向收益回填、健康检查、自进化优化。

说明：

- 健康度页面的普通查看不会写入历史；运行信号、每日检查、完整闭环才会写健康历史。
- 信号收益基于本地日线数据回填，数据未成熟时会显示为空。
- 优化候选不会自动覆盖原策略，必须人工晋级。
- 晋级会写入策略版本快照，并更新原策略规格。
- 智能策略回测已计入默认交易成本、滑点、涨跌停买卖限制、换手率和基准收益，但仍不等于实盘表现。

## 6. 推荐日常使用流程

### 6.1 盘后分析流程

```text
数据同步 -> 数据质量检查 -> 市场环境 -> 选股器 -> 分析驾驶舱
-> 回测验证 -> 交易计划 -> 预警 -> 模拟记录 -> 复盘
```

具体步骤：

1. 在“数据同步”执行增量同步。
2. 在“数据质量”确认质量等级和异常。
3. 在“市场环境”确认市场状态。
4. 在“选股器”扫描候选股票。
5. 在“分析驾驶舱”查看单票方向、价格结构和风险。
6. 在“回测中心”验证候选组合或策略。
7. 在“交易计划”记录触发价和失效价。
8. 在“预警中心”或计划绑定预警。
9. 在“模拟组合”记录模拟成交。
10. 在“复盘中心”记录盈亏、错误和经验。

### 6.2 快速体验流程

如果只想快速体验，不做全量数据初始化：

1. 启动后端和前端。
2. 打开分析驾驶舱。
3. 输入 `000001` 或 `600519`。
4. 点击查询并运行分析。
5. 查看 K 线、方向卡和支撑压力。

这种方式适合体验单标的分析，但批量选股和回测可能因本地数据不足而不完整。

### 6.3 数据维护流程

建议：

- 每个交易日盘后执行增量同步。
- 每周查看一次数据质量页面。
- 如果出现大量缺失或 D 级数据，执行全量续传。
- 如果公开数据源异常，稍后重试或调整数据源顺序。

## 8. 常见问题

### 8.1 前端提示后端未连接

检查后端是否启动：

```bash
curl http://localhost:8000/health
```

如果没有返回健康状态，重新启动后端：

```bash
cd backend
python start_server.py
```

### 8.2 查询股票失败

可能原因：

- 股票代码错误。
- 网络无法访问公开数据源。
- 数据源临时不可用。
- 本地缓存损坏。

处理建议：

- 确认代码为 6 位数字。
- 切换或调整数据源顺序。
- 清理缓存后重试。
- 稍后重新同步。

### 8.3 选股器扫描结果很少

可能原因：

- 本地 K 线尚未同步。
- 开启了“仅本地 K 线”，无缓存股票被跳过。
- 筛选条件过严。
- 多个预设交集导致命中减少。

处理建议：

- 先执行全量或增量同步。
- 减少预设数量。
- 扩大扫描板块。
- 使用自定义股票池验证。

### 8.4 回测结果为空或异常

可能原因：

- 输入股票没有足够历史数据。
- 起止日期内数据缺失。
- 策略没有产生交易信号。
- 数据质量较差。

处理建议：

- 先在数据质量页面查询对应股票。
- 增加回测标的数量。
- 切换策略。
- 重新同步日 K。

### 8.5 K 线图为空

可能原因：

- 后端未返回 OHLCV。
- 股票代码无效。
- 数据源临时失败。
- 时间周期内无数据。

处理建议：

- 切换周期为 1 年或全部。
- 点击刷新。
- 检查后端日志。
- 使用 Swagger 调用 `/api/v1/stocks/{symbol}` 验证。

### 8.6 预警没有触发

检查项：

- 预警规则是否启用。
- 阈值是否合理。
- 是否处于冷却时间内。
- 后端服务是否持续运行。
- 对应股票是否能拉到最新数据。
